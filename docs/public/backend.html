Chapter 00 · The brief

Three apps, one core.

A backend for a basic e-hailing system — rider, driver, and admin — built as a modular monolith on Postgres and Redis. One backend, one database, three frontends. Designed to ship fast today and split cleanly later.

Chapter 01 · Topology

The shape of the system, at a glance.

Three frontends speak to one backend over versioned, audience-scoped API prefixes. The backend persists to one Postgres database and leans on Redis for hot real-time state.

Chapter 05 · House rules

Eight rules to keep the monolith honest.

A modular monolith degrades into a regular monolith the day someone reaches across a boundary "just this once". These are the fences.

01

One service per module is the source of truth.

Routes, workers, and websockets all delegate to the module's service. Repos stay private to their module — no cross-module SQL.

02

Routes by audience, domain by context.

api/{rider,driver,admin}/ hold the per-audience surface; modules/ hold the domain. Routes are thin and call into services. DTOs live with routes; domain types live with modules.

03

Trip state changes go through the state machine.

No raw UPDATE trips SET status. Every transition writes to trip_events — the audit log doubles as a replay source.

04

Dispatch and tracking have no HTTP routes.

Dispatch is queue-driven (BullMQ); tracking is WebSocket-driven (Redis GEO). Different scaling profiles, different transports.

05

Workers run in a separate process.

The API and the worker share code, not a runtime. Scale where it's needed without touching request latency.

06

Money in integer cents, geometry in 4326.

amount_cents int · GEOGRAPHY(POINT, 4326). Floats compound rounding errors; settled.

07

One Postgres database, three roles.

app_owner for migrations. app_user at runtime. app_readonly for BI. Auth lives in middleware — the backend is the trust boundary.

08

Design for extraction, not separation.

Modules look like services from outside (one entrypoint, owned data) so any one — likely tracking first — can be lifted into its own process when it earns it.
